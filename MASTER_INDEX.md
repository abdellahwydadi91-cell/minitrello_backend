### 1️⃣ JWT Token
**Explication simple** :
> "Le JWT est un token signé généré au login. À chaque requête protégée, le frontend l'envoie en header Authorization. Le serveur valide la signature et accepte la requête."

**Où en parler** : Explication sécurité  
**Document** : GUIDE_SOUTENANCE_BACKEND.md section 2

### 2️⃣ Rôles (CHEF_PROJET / MEMBRE)

**Où en parler** : Contrôle d'accès  
**Document** : FEUILLE_TRICHE_SOUTENANCE.md

## 🎯 RÉPONSES PRÊTES AUX QUESTIONS FRÉQUENTES

### Q: Pourquoi Spring Boot ?
**R** : Framework robuste pour les APIs REST, gestion automatique injection, intégration Spring Security

### Q: Pourquoi JWT ?
**R** : Stateless, scalable, idéal pour APIs REST et SPA frontend

### Q: Comment gérez-vous la sécurité ?
**R** : BCrypt pour mots de passe, JWT + signature HMAC pour tokens, vérification rôles, CORS configuré

### Q: Comment implémentez le drag & drop ?
**R** : Position entière sur chaque élément, deux endpoints pour déplacement horizontal/vertical

### Q: Comment testez-vous ?
**R** : Tests unitaires avec JUnit5, integration tests avec MockMvc, Swagger UI pour tests manuels
